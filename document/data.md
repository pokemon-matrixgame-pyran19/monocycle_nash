# このファイルについて

## 概要
同様の初期値に対する分析を繰り返しながら少しずつ分析対象を広げていくことになる。
共通する条件が多くなり、いくつかの条件はセットで変更しないと意味がないことから意味のあるまとまり毎に共通化して識別子で指定できると便利である。本ドキュメントではそれらのデータの読み込みフォーマットについて記述

## フォーマット

data/(クラスの種類)/(識別子)フォルダに実験条件を永続化して管理しやすくする。形式はtoml.

data.mdをルートとした木構造の入力値とする。何もなければ単一ファイルで完結、$refが含まれるノードがあればそのノードを$refの参照先のtomlファイルで置換する。

これにより下位構造のみの実験条件変更に対応しやすくなる。


## 例

クラスSpamを次のように初期化するデータを保持したいとする。x=1,y=3のデータを他の初期値のデータと区別してjankenと呼ぶとする.

spam=Spam(x=1,y=3, sub_state=Egg(z=2,w=6))

この時、次のようにフォルダを作る

data/spam/janken/data.md

``` data.toml
x = 1
y = 3
[sub_state] 
z = 2
w = 6
```
次のように分割して収納することも可能にする.
data/spam/janken/---
    |--- data.md
    |--- sub_state/
        |---nash.md
        |---other.md

この時は次のように書く

``` data.toml
x = 1
y = 3
[sub_state] 
$ref="nash"
```
子ファイルの構造は以下
``` nash.toml
x = 1
y = 3
[sub_state] 
z = 1
w = 1
```
