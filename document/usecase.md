# プログラムのユースケース

入力として利得行列を指定するケース(ほぼすべてのケースだが)は共通で、単相性モデルでキャラクターを指定される場合も行列を直接配列で受け取る場合も、キャラクターから構築利得行列を生成する場合も対応する。入力値に応じてそれぞれの方法で利得行列を生成した後、下記のユースケースに合わせて分岐

1. 利得行列Aに対して利得行列を表す有効グラフを出力
2. 利得行列Aを与えてそれに対する均衡戦略や純粋戦略や乖離度を出力
3. 複数キャラクターを配列として受け取り、相性ベクトルをプロット
4. 二つの利得行列A,Bを入力として比較(近似の妥当性チェック). 均衡戦略や環境乖離度を指標にして評価
5. ランダム生成した入力値から利得行列を作ってそれらに対して近似等の妥当性を確率的に評価。(近似の妥当性)

4,5に関しては将来的な機能なのでしばらくの間は設計時に考慮しない
