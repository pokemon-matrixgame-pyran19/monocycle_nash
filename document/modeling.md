# データのモデリング

プログラムに使用されるデータと、プログラムで実行される内容を整理

## エンティティ

1. 利得行列  →環境を定義。常に存在。 
2. 戦略 -> 利得行列の各行、各列に対応. キャラクター or 構築のいずれかを使用
2. キャラクター  
    - ラベル   可視化の時に使うラベル。ここまでは常にある
    - パワー、相性 → 一段階近似したものなのであったりなかったり。原点の取り方で表面的には変化。
3. 構築 -> 複数キャラクターの組からなるチームを構築と呼ぶ。構築のゲームの利得行列はキャラクターのゲームの値を利得とするものになる。
4. visualizer -> 結果の数値をグラフなどで可視化
5. 使用率(混合戦略) -> 合計100%になるようにキャラクター(戦略)に確率を分配.  

## アプリケーション

利得行列 <- キャラクター(パワー、相性)  :特定の近似理論のもと利得行列を導出
均衡解 <- 利得行列 : 線形最適化でいつでも可。
ε戦略 <- 利得行列, 使用率 
構築利得行列 <- キャラクター利得行列  :  キャラクターゲームのゲーム値を利得
構築順 <- キャラクター順 : 利得行列の表現は戦略の並び順で異なる。並び順を固定するための関数化
原点移動・等パワー座標 <- キャラクター (パワー、相性ベクトル)の平行移動任意性を使って複数キャラクターを等パワーにする座標を見つける
